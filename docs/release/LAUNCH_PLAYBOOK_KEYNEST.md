# KeyNest Launch Playbook

## Goal
個人ユーザーの初期流入を作りつつ、法人導入の相談導線を同時に立ち上げる。

## Phase 1: 公開前
1. TestFlight で iOS 実機導線を確認
2. Google Play 側のアカウント確認を完了
3. スクリーンショット 5 枚を作成
4. ストア文言を反映
5. LP を公開

## Phase 2: 公開直後 7 日間
1. App Store / Play を公開
2. X で告知
3. Product Hunt 投稿
4. Reddit / Hacker News に投稿
5. GitHub README に導線を追加
6. 既存人脈へ 10 件連絡

## Phase 3: 公開後 30 日
1. 初期ユーザーの定着を見る
2. よく使われる機能を把握
3. 企業向け相談フォームを追加
4. 3 社以上にヒアリング
5. Team / Business プラン案を定義

## KPI
- 初週ダウンロード数
- 初週継続率
- QR 登録完了率
- Push 承認利用率
- 復元利用率
- 法人問い合わせ数

## Messaging split
### 個人向け
- 認証コード、Push 承認、復元を一つに
- 機種変更後も戻しやすい

### 企業向け
- 軽量に導入できる認証アプリ
- 小規模チームでも始めやすい
- 将来的なブランド差し替えにも向く
