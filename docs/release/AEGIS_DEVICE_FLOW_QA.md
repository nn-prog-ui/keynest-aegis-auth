# Nemokey 実機導線 QA

対象導線:
1. 追加
2. QR登録
3. コード表示
4. Push承認
5. クラウド復元

## 事前準備
- iOS/Android 実機にデバッグビルドをインストールする
- `KEYNEST_API_URL` にAPIサーバーURLを設定する
- Push検証時は Firebase/APNs 設定済みであること

## チェック手順
1. 初回起動で `Welcome to Nemokey` が表示されること
2. `Get started` でホームに遷移すること
3. `アカウント追加` -> `QRコードを読み取る` から `otpauth://` を登録できること
4. 追加直後にコードカードへ表示され、残り秒ゲージが進むこと
5. `承認` タブで `テストリクエスト生成` を押し、`Approve` / `Deny` が反映されること
6. `クラウドバックアップ` で保存し、バックアップIDが表示されること
7. 既存アカウントを削除し、同じバックアップID + 合言葉で復元できること
8. サーバー停止時に保存/復元がローカルフォールバックで成功すること
9. アプリをバックグラウンド->復帰すると端末認証ロックが表示されること

## 既知の判定
- `接続できなかったため、この端末の安全領域に保存しました` が出た場合はローカルフォールバック成功
- `オフライン保存データから復元しました` が出た場合はローカルフォールバック復元成功

## 合格条件
- 上記 1-9 が全て再現し、クラッシュ/フリーズがないこと
